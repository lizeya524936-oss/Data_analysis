# 传感器 Hill 方程拟合分析工具

**版本：v1.4.1** | 适用平台：Windows / macOS / Linux

---

## 工具简介

本工具专为多传感器压力实验数据设计，核心分析流程如下：

1. 每个 CSV 文件对应**一次实验**，包含同一组 4 个传感器的采集数据
2. 对每行的 4 个传感器值**逐行求和**，得到该次实验的"求和曲线"
3. 将多次实验的求和曲线按压力分箱取**均值**，得到"平均曲线"
4. 对平均曲线进行 **Hill 方程**和**双曲线**拟合
5. 将拟合曲线回代到每次实验的求和曲线，计算**残差**

工具提供两种工作模式：**拟合模式**（自动拟合参数）和**加载参数模式**（使用已有参数直接计算残差），适用于跨批次数据一致性验证。

## 数据格式要求

### CSV 文件格式

每个 CSV 文件代表**一次完整实验**，需包含以下列：

- **第 1 列**：序号或时间戳（可忽略）
- **第 2 列**：压力值（列名含 `pressure`、`压力`、`force`、`load` 之一）
- **第 3 列起**：各传感器数值（列名含 `sensor`、`传感器`、`#数字`、`ch数字` 之一）

### 示例表头

```
index,pressure,sensor#1,sensor#2,sensor#3,sensor#4
1,0.5,12.3,11.8,13.1,12.6
2,1.2,25.4,24.9,26.2,25.8
...
```

### 注意事项

- 文件编码支持 UTF-8（含 BOM）和 GBK
- 每次实验上传**一个文件**，多次实验上传**多个文件**
- 建议每批次上传 3～10 个文件以保证均值曲线的统计可靠性

## 界面说明

### 左侧控制面板

#### 数据文件区

点击「**+ 添加文件**」按钮选择 CSV 文件，支持批量选择。文件列表显示每个文件识别到的传感器数量。点击「**清空**」移除所有已加载文件。

#### 分析配置

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 压力范围 (N) | 只分析该范围内的数据点 | 0 ~ 200 |
| 求和值最小阈值 | 过滤静息期（求和值低于此值的行被忽略） | 0 |
| 平滑窗口大小 | 均值曲线的移动平均窗口，越大越平滑 | 5 |

#### 离群点剪除

勾选「**启用离群点剪除**」后，可设置阈值（默认 ±15%）。在每个压力分箱内，偏离该分箱均值超过设定百分比的数据点将被剔除，有助于减少偶发噪声对均值曲线的影响。

#### Hill 拟合初始值（可选）

若自动拟合结果不理想，可手动输入 a、b、n 的初始猜测值，引导优化器收敛到正确解。

#### 加载拟合参数区域（v1.4 新增）

用于**加载参数模式**，不重新拟合，直接用已有参数计算残差。

- **手动输入**：在 a / b / n 输入框直接填写 Hill 参数；双曲线参数（a、b）为可选项
- **从 JSON 导入**：点击「📂 从 JSON 导入参数」，选择之前导出的 JSON 文件，参数自动填入
- **执行计算**：点击「▶ 加载参数计算残差」，跳过拟合步骤直接输出残差结果

#### 拟合参数展示区

显示当前分析的所有参数和指标，包括：

- Hill 方程参数 a、b、n 及均值曲线 RMSE / R²
- 双曲线参数 a、b 及均值曲线 RMSE / R²
- 各次实验回代的 RMSE 和 R²
- 加载参数模式的对应指标（若已运行）

#### 导出结果

支持导出为 **CSV** 和 **JSON** 两种格式，包含所有拟合参数和回代指标。

#### Tab 1：拟合曲线

展示所有实验的求和曲线（彩色半透明线）、均值曲线（白色）、Hill 拟合曲线（红色实线）和双曲线拟合（青色虚线），图例中包含参数值和 R²。

#### Tab 2：残差分析

三层布局：

## 典型使用流程

### 流程一：标准拟合分析

```
1. 点击「+ 添加文件」→ 选择同一批次的多个 CSV 文件
2. 设置压力范围（根据实验设计，如 0 ~ 150 N）
3. 可选：启用离群点剪除（建议阈值 15%～20%）
4. 点击「▶ 开始拟合分析」
5. 查看「拟合曲线」Tab 确认拟合质量（R² > 0.99 为良好）
6. 查看「残差分析」Tab 评估各次实验一致性
7. 点击「导出 JSON」保存参数供后续使用
```

### 流程二：跨批次一致性验证（加载参数模式）

```
1. 加载新批次的 CSV 文件
2. 在「加载拟合参数」区域点击「📂 从 JSON 导入参数」
   → 选择上一批次导出的 JSON 文件
3. 点击「▶ 加载参数计算残差」
4. 查看「加载参数残差」Tab
   → RMSE 和 R² 与原批次接近，说明传感器响应一致
   → 偏差较大，说明传感器特性发生漂移
```

## 拟合方程说明

### Hill 方程

```
y = a * x^n / (b^n + x^n)
```

| 参数 | 含义 |
|------|------|
| `a` | 饱和值（最大响应，单位与传感器求和值相同） |
| `b` | 半饱和压力（达到最大响应一半时的压力，单位 N） |
| `n` | Hill 系数（曲线陡峭程度，n=1 退化为双曲线） |

### 双曲线（Michaelis-Menten）

```
y = a * x / (b + x)
```

即 Hill 方程在 n=1 时的特例，参数含义与 Hill 方程相同。

## 常见问题（FAQ）

**Q：导入 JSON 时提示"未找到有效的 Hill 参数"？**

A：v1.4.1 已修复此问题。工具现在兼容本工具所有版本导出的 JSON 格式（`fit_mode` 嵌套格式、`loaded_param_mode` 格式、简单格式、扁平格式）。如仍报错，错误弹窗会显示 JSON 的实际顶层键名，可据此判断文件是否损坏。

**Q：拟合结果 R² 偏低（< 0.95）怎么办？**

A：建议依次尝试：① 调整压力范围，排除静息期数据；② 启用离群点剪除；③ 手动输入更接近实际的初始值（a 约为最大求和值，b 约为曲线中点压力，n 从 0.8～1.2 开始尝试）。

**Q：程序提示"有效数据不足"？**

A：通常是压力范围设置过窄，或求和值阈值设置过高，导致大量数据被过滤。请检查原始 CSV 数据的压力范围，适当放宽设置。

**Q：如何处理传感器列名不标准的文件？**

A：工具会自动识别列名中含有 `sensor`、`传感器`、`#数字`、`ch数字` 的列作为传感器列。若列名完全不符合规则，工具会将第 3 列起的所有列视为传感器列。建议将列名修改为标准格式后再导入。

**Q：可以只分析 1 个文件吗？**

A：可以，但建议至少 3 个文件以上，均值曲线才具有统计意义。单文件时"均值曲线"即为该文件本身的求和曲线。
