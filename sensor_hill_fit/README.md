# 传感器 Hill 方程拟合分析工具

**版本：v1.8.1** | 适用平台：Windows / macOS / Linux

---

## 工具简介

本工具专为多传感器压力实验数据设计，核心分析流程如下：

1. 每个 CSV 文件对应**一次实验**，包含同一组传感器的采集数据
2. 对每行的多个传感器值**逐行求和后取平均**（除以传感器数量），得到该次实验的"单传感器均值曲线"
3. 将多次实验的均值曲线按压力分箱取**均值**，得到"平均曲线"
4. 对平均曲线进行 **Hill 方程**和**双曲线**拟合
5. 将拟合曲线回代到每次实验的均值曲线，计算**残差**
6. 对多次实验进行**一致性评估**（关键压力点 CV 分析 + 残差分布统计）

工具提供三种工作模式：

| 模式 | 入口 | 适用场景 |
|------|------|---------|
| **拟合模式** | 「▶ 开始拟合分析」 | 首次建立传感器响应模型，自动拟合 Hill/双曲线参数 |
| **加载参数模式** | 「▶ 加载参数计算残差」 | 用已有参数验证新批次数据，不重新拟合 |
| **反向推算模式** | 「🔄 计算反推压力」 | 已知 ADC 值，反推对应压力 |

## 数据格式要求

### CSV 文件格式

每个 CSV 文件代表**一次完整实验**，需包含以下列：

- **第 1 列**：序号或时间戳（可忽略）
- **第 2 列**：压力值（列名含 `pressure`、`压力`、`force`、`load` 之一）
- **第 3 列起**：各传感器数值（列名含 `sensor`、`传感器`、`#数字`、`ch数字` 之一）

### 示例表头

```
index,pressure,sensor#1,sensor#2,sensor#3,sensor#4
1,0.5,12.3,11.8,13.1,12.6
2,1.2,25.4,24.9,26.2,25.8
...
```

### 注意事项

- 文件编码支持 UTF-8（含 BOM）和 GBK
- 每次实验上传**一个文件**，多次实验上传**多个文件**
- 建议每批次上传 3～10 个文件以保证均值曲线的统计可靠性
- 默认分析压力范围为 **0–100N**

## 界面说明

### 左侧控制面板

#### 数据文件区

点击「**+ 添加文件**」按钮选择 CSV 文件，支持批量选择。文件列表显示每个文件识别到的传感器数量。点击「**清空**」移除所有已加载文件。

#### 分析配置

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 压力范围 (N) | 只分析该范围内的数据点 | 0 ~ 100 |
| 求和值最小阈值 | 过滤静息期（均值低于此值的行被忽略） | 0 |
| 平滑窗口大小 | 均值曲线的移动平均窗口，越大越平滑 | 5 |

#### 离群点剪除

勾选「**启用离群点剪除**」后，可设置阈值（默认 ±15%）。在每个压力分箱内，偏离该分箱均值超过设定百分比的数据点将被剔除，有助于减少偶发噪声对均值曲线的影响。

#### Hill 拟合初始值（可选）

若自动拟合结果不理想，可手动输入 a、b、n 的初始猜测值，引导优化器收敛到正确解。

#### 加载拟合参数区域

用于**加载参数模式**，不重新拟合，直接用已有参数计算残差。

- **手动输入**：在 a / b / n 输入框直接填写 Hill 参数；双曲线参数（a、b）为可选项
- **从 JSON 导入**：点击「📂 从 JSON 导入参数」，选择之前导出的 JSON 文件，参数自动填入
- **执行计算**：点击「▶ 加载参数计算残差」，跳过拟合步骤直接输出残差结果

#### ADC → 压力反向推算区域

基于已有 Hill 拟合参数，输入 ADC 值反推对应压力。

- **ADC 输入**：支持多个值（逗号、空格或换行分隔）
- **参数来源**：可选「拟合结果」或「加载参数」
- **压力范围限制**：反推结果限制在 0–100N 范围内，超出范围会给出警告
- **反向公式**：`x = b · (y / (a - y))^(1/n)`，其中 y 为 ADC 值，x 为压力

#### 拟合参数展示区

显示当前分析的所有参数和指标，包括：

- Hill 方程参数 a、b、n 及均值曲线 RMSE / R²
- 双曲线参数 a、b 及均值曲线 RMSE / R²
- 各次实验回代的 RMSE 和 R²
- 加载参数模式的对应指标（若已运行）
- **一致性评估结果**：关键压力点 CV 详情 + 各次实验残差统计

#### 导出结果

支持导出为 **CSV** 和 **JSON** 两种格式，包含所有拟合参数和回代指标。

## 一致性评估说明

### 维度三：关键压力点一致性（CV 分析）

**原理：** 对每次实验独立拟合 Hill 参数，然后在 7 个关键压力点（5/10/20/30/50/70/100N）计算各自的 ADC 预测值，再计算每个压力点的**变异系数 CV**（标准差 / 均值 × 100%）。

**CV 判据：**

| CV 范围 | 等级 | 含义 |
|---------|------|------|
| ≤ 3% | 优秀 | 多次实验高度一致，传感器重复性极好 |
| 3%–5% | 合格 | 正常工程可接受范围 |
| 5%–8% | 警告 | 一致性偏差较大，需关注 |
| > 8% | 不合格 | 传感器批次间差异过大或存在异常实验 |

**评分规则：** 平均 CV ≤ 3% 得 100 分，≥ 8% 得 0 分，中间线性插值。

### 维度五：残差分布

**原理：** 收集所有实验的回代残差（实际值 - 预测值），计算总体标准差 σ，并分析各次实验的残差统计特征（均值、标准差、最大绝对残差、偏度）。

**评分规则：** σ ≤ 5 得 100 分，σ ≥ 30 得 0 分，中间线性插值。

## 典型使用流程

### 流程一：标准拟合分析

```
1. 点击「+ 添加文件」→ 选择同一批次的多个 CSV 文件
2. 设置压力范围（默认 0 ~ 100 N）
3. 可选：启用离群点剪除（建议阈值 15%～20%）
4. 点击「▶ 开始拟合分析」
5. 查看「拟合曲线」Tab 确认拟合质量（R² > 0.99 为良好）
6. 查看「残差分析」Tab 评估各次实验一致性
7. 查看「一致性评估」Tab 查看关键压力点 CV 和残差分布
8. 点击「导出 JSON」保存参数供后续使用
```

### 流程二：跨批次一致性验证（加载参数模式）

```
1. 加载新批次的 CSV 文件
2. 在「加载拟合参数」区域点击「📂 从 JSON 导入参数」
   → 选择上一批次导出的 JSON 文件
3. 点击「▶ 加载参数计算残差」
4. 查看「加载参数残差」Tab
   → RMSE 和 R² 与原批次接近，说明传感器响应一致
   → 偏差较大，说明传感器特性发生漂移
```

### 流程三：ADC → 压力反向推算

```
1. 先完成拟合分析（或加载已有参数）
2. 在「ADC → 压力反推」区域输入 ADC 值（支持多个，逗号分隔）
3. 选择参数来源（拟合结果 / 加载参数）
4. 点击「🔄 计算反推压力」
5. 查看「反向推算」Tab 的正向/反向曲线和查询点标注
```

## 拟合方程说明

### Hill 方程（正向：压力 → ADC）

```
y = a · x^n / (b^n + x^n)
```

| 参数 | 含义 |
|------|------|
| `a` | 饱和值（最大响应，单位与传感器均值相同） |
| `b` | 半饱和压力（达到最大响应一半时的压力，单位 N） |
| `n` | Hill 系数（曲线陡峭程度，n=1 退化为双曲线） |

### Hill 方程（反向：ADC → 压力）

```
x = b · (y / (a - y))^(1/n)
```

约束条件：0 < y < a，且反推结果限制在 0–100N 范围内。

### 双曲线（Michaelis-Menten）

```
y = a · x / (b + x)
```

即 Hill 方程在 n=1 时的特例，参数含义与 Hill 方程相同。

## 常见问题（FAQ）

**Q：导入 JSON 时提示"未找到有效的 Hill 参数"？**

A：v1.4.1 已修复此问题。工具现在兼容本工具所有版本导出的 JSON 格式（`fit_mode` 嵌套格式、`loaded_param_mode` 格式、简单格式、扁平格式）。如仍报错，错误弹窗会显示 JSON 的实际顶层键名，可据此判断文件是否损坏。

**Q：拟合结果 R² 偏低（< 0.95）怎么办？**

A：建议依次尝试：① 调整压力范围，排除静息期数据；② 启用离群点剪除；③ 手动输入更接近实际的初始值（a 约为最大均值，b 约为曲线中点压力，n 从 0.8～1.2 开始尝试）。

**Q：程序提示"有效数据不足"？**

A：通常是压力范围设置过窄，或均值阈值设置过高，导致大量数据被过滤。请检查原始 CSV 数据的压力范围，适当放宽设置。

**Q：如何处理传感器列名不标准的文件？**

A：工具会自动识别列名中含有 `sensor`、`传感器`、`#数字`、`ch数字` 的列作为传感器列。若列名完全不符合规则，工具会将第 3 列起的所有列视为传感器列。建议将列名修改为标准格式后再导入。

**Q：可以只分析 1 个文件吗？**

A：可以，但建议至少 3 个文件以上，均值曲线才具有统计意义。单文件时"均值曲线"即为该文件本身的均值曲线。一致性评估需要至少 2 个文件。

**Q：一致性评估中 CV 值如何解读？**

A：CV（变异系数）= 标准差 / 均值 × 100%，是衡量多次实验在同一压力点的相对离散程度。CV ≤ 3% 为优秀，3%–5% 为合格，5%–8% 需关注，> 8% 为不合格。通常低压区（5–10N）CV 较高，高压区（70–100N）CV 较低。

**Q：反向推算结果显示"超出 100N 范围"？**

A：说明输入的 ADC 值对应的压力超过了 100N 的有效量程。可以参考数值但需注意此时已超出标定范围，精度无法保证。

## 核心代码模块

本工具的核心算法已拆解为独立的 `hill_core` 模块包，位于 `hill_core/` 目录下，可直接导入到其他 Python 项目中使用：

| 模块 | 文件 | 职责 |
|------|------|------|
| 数学模型 | `models.py` | Hill 正向/反向公式、双曲线、评估指标 |
| CSV 解析 | `parser.py` | 自动识别压力列和传感器列 |
| 数据预处理 | `preprocessing.py` | 传感器求和均值、分箱、平滑 |
| 拟合引擎 | `fitting.py` | Hill/双曲线非线性拟合 |
| 残差分析 | `residuals.py` | 回代残差计算 |
| 导入导出 | `io_utils.py` | JSON/CSV 序列化 |
| 流水线 | `pipeline.py` | 一键分析入口 |

详细接口说明见 `hill_core/README.md` 和 `Hill_Fitting_Technical_Spec.md`。
